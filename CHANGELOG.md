## [v1.5.0] - 2019-09-03

### :notepad_spiral: Changes

- Fixed a few Python 3.x nits in the parser code
- Changed generated anchor slugs to not collide with IDs used by xml2rfc, which in v3 mode reserves certain prefixes for section, figure, and table anchors; also fixed an xml2rfc v2v3 converter compatibility issue
- Added outdent to id2xml progress statements to more clearly differentiate them from warnings and errors
- Added the possibility of setting document stream, IPR, and consensus values with switches instead of deriving them from the 'Status of this Memo' section
- Added a list of valid submission types and stream names to `id2xml.utils`
- Tweaked several command-line options to better match those of xml2rfc; added help for trace options when `--help` is given with `--debug`

---

## [v1.4.4] - 2018-10-17

### :notepad_spiral: Changes

- Tweaked comment insertion to avoid inserting `--` inside HTML comments, which is not permitted and could cause failures when id2xml inserts conversion-error comments into generated XML
- Added code to look for Abstract both before and after Status-of-Memo and Copyright sections (placement changed around RFC 5550)

---

## [v1.4.3] - 2018-10-12

### :notepad_spiral: Changes

- Fixed some issues with author recognition and an infinite loop
- Added handling for the boilerplate grammar change 'a candidate' → 'candidates' in the Status of Memo section
- Aligned the short options for verbose and version with common practice

---

## [v1.4.2] - 2018-07-19

### :notepad_spiral: Changes

- Fixed a couple of issues that prevented processing of rfc7752
- Aligned generator comment date format with that used by xml2rfc
- Added a 'Generated by' comment showing id2xml version and time of run

---

## [v1.4.1] - 2017-12-19

### :notepad_spiral: Changes

- Deal better with comma-separated lists of RFC numbers in `Updates:` and `Obsoletes:` fields in the input document header
- Added warnings as comments inline in the XML file (in addition to showing them on stderr)

---

## [v1.4.0] - 2017-11-02

### :notepad_spiral: Changes

- Added support for tables with `style='all'`

## [v1.3.1] - 2017-10-30

### :notepad_spiral: Changes

- Added http/https normalization when comparing boilerplate; tweaked punctuation in some warning messages for consistency; permitted some additional variation in appendix section numbering
- Added a new test document containing https:// boilerplate URLs
- Added Glen and Matt @amsl.com to the release-mail recipients

## [v1.3.0] - 2017-09-23

### :notepad_spiral: Changes

- Changed a dependency from `pathlib` to the recommended replacement `pathlib2`, and added a version requirement to the xml2rfc dependency
- Added support for id2xml conversion to xml2rfc vocabulary version 3 format via the v2v3 conversion component of xml2rfc, invoked with a new switch: `-3` / `--schema-v3`

## [v1.2.3] - 2017-09-23

### :notepad_spiral: Changes

## [v1.2.2] - 2017-08-10

### :notepad_spiral: Changes

- Added a warning when an author name is seen in the document title abbreviation

## [v1.2.1] - 2017-08-10

### :notepad_spiral: Changes

## [v1.2.0] - 2017-08-09

### :notepad_spiral: Changes

- Added recognition of additional variations on the title of the copyright section, based on common variations received from the RFC Editor
- Fixed a bug in the handling of docname parts in reference text when the docname cannot be split into series info

---

## [v1.1.0] - 2017-07-27

### :notepad_spiral: Changes

- Added space normalization when checking/skipping boilerplate text
- Added support for using reference anchors to identify references that may have bibxml entries, including W3C, IEEE and 3GPP references; tweaked the reference organization pattern; some refactoring
- Tweaked the help text for trace switches
- Removed the dependency on `pyterminalsize`, but will use it if installed

---

## [v1.0.3] - 2017-07-01

### :notepad_spiral: Changes

- Added recognition of series info of the form 'RFC1234' in addition to 'RFC 1234'; added recognition of 'Internet Draft' on the first page left column in addition to 'Internet-Draft'; added guidance to the reference parsing failure message; added another reference pattern
- Tweaked the section number regexp
- Added acceptance of day of month in footer line date, which also improves extraction of short title
- Be more permissive regarding blank lines in the author address format

---

## [v1.0.2] - 2017-06-18

### :notepad_spiral: Changes

- Set the full organization name in the author element from the authors' addresses section, using what was found on the first page (if different) as the `abbrev` attribute
- Changed handling of RFC quotations with numbers below 1000 to always use zero-padded numbers matching the bibxml libraries, inserting entity references instead of in-place entries
- Changed text-table identification to not attempt to handle one-row tables as texttables

---

## [v1.0.1] - 2017-06-14

### :notepad_spiral: Changes

- Added generation of a `sortrefs` PI matching whether the original's RFC references were sorted or not
- Tweaked the slugifier and applied it to `section-*` anchors to ensure validity; fixed an issue causing trailing commas in entity names
- Rewrote back matter handling to permit the various back sections to occur in any order; added more 'work in progress' variants; added new reference patterns; removed the expectation that references have a terminating period
- Refined header and footer stripping to consider end-of-line commas and to require short lines triggering paragraph breaks to contain text
- Fixed a bug in line reading that could cause the first line of a document to be skipped; added recognition of additional Standards Track status indications; fixed mismatched authors between first page and Author's Addresses section; refined header/footer stripping for additional variations
- Fixed a number of places where `warn()` was called with a line object instead of the line number
- Changed to using the supplied figure or texttable label to set the `title` attribute when rendering a figure as texttable or vice versa
- Refined the tokenizer for the text parser to correctly handle constructs like `(Section N.N)`
- Eliminated trailing blank cells in texttables
- Added RFC-Editor staff to the release notification list

---

## [v1.0.0] - 2017-05-30

### :notepad_spiral: Changes

- Split functionality into separate `run.py`, `parser.py`, and `utils.py` files; adjusted Makefile and MANIFEST accordingly
- Reference section entries are now entity references for drafts and RFCs instead of in-place generated XML
- Slight refactoring of how `reference_anchors` and `section_anchors` lists are generated
- Added `xref` elements for `Section N.nn` strings that reference document sections
- Multiple rounds of refactoring to clean up and organise the code
- Cleaned up generated XML to avoid long lines and bunched tags
- Added a check on coupled debug trace switches (trace start requires trace stop)
- Further refined the regular expression which identifies code
- Refined header stripping to not join paragraphs where the first part has a short line
- Added more cases where list `hangIndent` is derived and set
- Added modification of the `text-list-symbols` PI to better match the source
- Improved the error message for missing stream information when processing older RFCs
- Fixed a bug in handling the XML tree for xrefs found in text interspersed with `vspace` elements
- Code optimisations
- Added the last two changelog sections to the release information shown on PyPI

---

## [v1.0.0-rc3] - 2017-05-26

### :notepad_spiral: Changes

- Committed updated (smaller) diff files for test baseline
- Added more alternatives to the code recognition regex, for XML tags and C statements
- Refined header/footer stripping to not join text broken across pages when there are too many intervening blank lines or when the last line is a table or figure label
- Added handling of blank lines in list items
- Added insertion of `subcompact` PIs for compact lists; fixed some warning message issues
- Added another comment delimiter to the code regex, applied to whole text blocks
- Moved list block normalisation functions into the `DraftParser` class; added recognition of compact lists; some refactoring
- Added more descriptive manpage text; tweaked manpage generation
- Added switches for trace start and stop on line number; renamed trace-related switches
- Refined `guess_list_style()`
- Added code to recognise 'centered' titles when they span the whole line
- Rewrote the top-left titlepage column parser to not assume any ordering of lines (exception: working group string, if present, must occur first)

---

## [v1.0.0-rc2] - 2017-05-22

### :notepad_spiral: Changes

- Tweaked the help text and manpage generation
- Updated MANIFEST and Makefile
- Added some missing files; updated the acceptable diffs in `test/ok/`

---

## [v1.0.0-rc1] - 2017-05-19

### :notepad_spiral: Changes

- Improved debug trace facilities with `--start-trace`, `--stop-trace`, and `--trace` options
- Improved the code recognition regex to handle more code and constant fragments as figures
- Added recognition and handling of `&amp;` and `&lt;` marks
- Added recognition of reference text date strings containing days
- Added recognition of another usage of 'Work-in-progress' in references
- Modified list handling to recognise lists in additional formats
- Added recognition of reference quotes within list text
- Added 2 new ways to recognize text to be captured as figures (recurring wide whitespace; text not being paragraph-filled)
- Added better handling of draft references for generating proper entity definitions in the doctype declaration
- Refined the test suite to show percentages of differing lines and to exclude ToC page number differences from diff linecounts
- Added support for title abbreviation in the footer rather than the header; created explicit title abbreviations for long titles with none available
- Don't interpret 'Internet-Draft' at the top left of the first page as a workgroup name
- Added list default style 'empty' (issue from julian.reschke@gmx.de)
- Added stripping of leading/trailing blanks from author name components (issue from julian.reschke@gmx.de)
- Renamed the subversion branch to match the selected tool name

---

## [v0.9.3] - 2017-05-15

### :notepad_spiral: Changes

- Added a `mkrelease` script
- Tweaked the test summary; corrected the revision number
- Fixed some bugs in the manpage generation; updated the manifest; cleaned up the repository content; bumped the version

## [v0.9.2] - 2017-05-15

### :notepad_spiral: Changes

- Added makefile rules for diff comparison tests between original and generated `.txt` files, and some baseline diffs
- Tweaked script message formatting slightly
- Added some config default values; tweaked command-line switches; generalized handling of recognized non-numbered section names; added recognition of additional reference formats; added per-function debug output methods toggleable from the command line
- Added support for `.id2xmlrc` config files and `symrefs='no'`; fixed a divide-by-zero bug; added `--debug` and `--quiet` options; changed conversion error messages to only show traceback with `--debug`; added handling of Figure and Table titles that only number without providing title text
- Added fixes for older documents and documents with many authors

---

## [v0.9.1] - 2017-05-10

### :notepad_spiral: Changes

- Fixed issues found for `draft-sparks-genarea-review-tracker-03.txt`: 'Work in Progress' annotation for non-draft references; code/figure text indented less than 3 spaces
- Fixed issues found for `rfc7842`: `None` instead of blank string as null input to `skip()`; failed to accept spelling 'acknowledgment'; failed to set attribute `numbered='no'` for unnumbered back matter sections
- Updated readme based on feedback from housley@vigilsec.com

---

## [v0.9.0] - 2017-05-09

### :notepad_spiral: Changes

- New readme text, more appropriate as PyPI description
- Increased the version number before pre-release; added an `upload` target to the makefile
- Fixed an issue with identifying URIs sections; tweaked the symbol ratio limit between figures and t/list items
- Rearranged id2xml as a package to enclose v2 and v3 `.rnc` and `.rng` files; added LICENCE, README, and MANIFEST files
- Changed argument parsing to use `argparse`; updated Makefile to process help text into manpage via `txt2man`
- Removed the local command-line script
- Moved command-line invocation code to `id2xml.run()`; added recognition of 'URIs' sections from erefs; added `setup.py` and supporting files for pip/setuptools installation
- Added a guard against overwriting potential XML files; added wrapping of warning and error messages
- Added `-o` (output file) and `-p` (output path) options; tweaked `--help` output
- Fixed issues with sublist identification, URL joining across lines, and wrong column settings when extracting table columns
- Improved eref support; some refactoring
- Added processing of erefs and xrefs in reference entries
- Stable point: parsing of front, back, and sections in place; generated XML produces text matching the original well when run through xml2rfc

---
